# Agentic 开发者实践

## 核心问题
当 AI Agent 让代码生成变得廉价，开发者应如何调整工作方式以最大化价值？

## 核心观点

> "代码廉价，但维护、支持和安全不廉价。"

Agent 时代的开发者不再是代码生产者，而是**判断者、引导者、架构师**。经验和品味的价值被放大，而非降低。

## 实践框架

### 🔄 开发节奏
| 实践 | 传统做法 | Agent 时代 |
|------|---------|-----------|
| 实现 | 谨慎，成本高 | 大胆实验，频繁重建 |
| 测试 | 单元测试为主 | 端到端测试优先 |
| 规格 | 项目前固定 | 随实现持续更新（[[living-specs]]） |
| 文档 | 结果文档 | 意图文档（why > what） |

### 🎯 精力分配
- **自动化**：所有简单、重复的任务
- **聚焦**：设计决策、性能优化、安全架构——真正难的问题
- **培养**：领域专业知识和用户理解（"品味"）

### 🧠 与 Agent 协作
- **经验放大效应**：技术深度越高，Agent 使用效率越高
- 能更精准地提示 Agent
- 能更快识别 Agent 的错误
- 能更有效地引导探索方向
